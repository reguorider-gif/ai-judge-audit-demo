价值故事

OpenCouncil 让你看见议院，AI Judge 让循环能被治理。

循环工程的关键不是让人继续点按钮，而是把人放到架构师位置：定义什么时候跑、什么时候停、谁来签字、下一轮能做什么。

01

接入一次真实循环

可以是 OpenCouncil 评议、RAG 回答、support agent 回复、coding-agent PR 或一次失败 eval。

02

拆开事实与信心

把每条 claim 对齐 source、judge disagreement、人工 verdict，避免“看起来合理”的循环继续扩大风险。

03

决定下一轮能否跑

不是只给分，而是输出 approve、rewrite、rerun、escalate 或 hold loop 的控制结论。

Core contract

产品形态不是页面，是可流转的 JSON。

首页负责讲清楚价值；JSON packet 才是真正能进入集成、review queue、GitHub issue 和未来 API 的操作面。

  • Input: trigger、output、sources、tool logs、judge scores。
  • Audit: claim ledger、evidence status、seat disagreement。
  • Decision: human verdict、safer rewrite、next-run control。
{
  "loop_id": "support-policy-enterprise-refund",
  "trigger": "customer_support_ticket",
  "audit": {

    "decision": "hold_loop",
    "reason": "Correct facts were turned into guarantees."
  },
  "next_run": {
    "action": "rewrite_then_rerun",
    "requires_human_gate": true
  }
}
证据路径

视觉现场在首屏，审计深度在报告里。

强视觉负责让人第一眼理解“这是一个真实循环”；样例报告和 JSON 负责证明它能落成 claim、source、verdict 和 next-run decision。

适配位置

放在团队已经使用的 eval、trace 和 agent 工具旁边。

AI Judge 不替代 eval stack。它把被选中的失败循环包装成产品、研究、客服、工程、合规都能读懂并接手的审计包。

Ragas, TruLens, DeepEval 把 groundedness / faithfulness failure 转成 claim-level audit packet。
Langfuse, Phoenix, Opik, LangSmith 给被选中的 trace 附上 reviewer-readable verdict 和 next-run decision。
Support / coding / research agents 阻止自动循环重复输出“很顺滑但不安全”的答案。

给我一个失败循环，换回一份审计包。

最低成本的验证就是一个真实 case：一条 trace、一个 RAG answer、一份 coding-agent PR、一次 support reply，或者一条失败 eval。
